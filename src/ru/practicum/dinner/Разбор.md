## 1. Приложение 1
> + #### Используем `switch case` с бесконечным циклом `while` в который будем передавать статические методы для вызова 
> + #### В самом классе `main` объявляем статические переменные классов который будут хранить значения `null` до вызова, а так же для полного доступа к статическим методом класс, так как они становятся переменными класса, а не конкретного объекта 
> + #### В методе main создаём объекты `scanner` и `dc` для того что бы точно знать когда они были созданы.
> + #### Все последующие методы тоже должны быть статическими, для того что бы метод `main` мог использовать их, без необходимости создавать объект `main`
> + #### Метод `printMenu` - а то-есть 1 пункт нашего консольного меню выводит список действий, для пользователя
> + #### Метод `addNewDish` нужен для того что бы пользователь мог добавить (Тип) блюда и (Названия) блюд этого типа.
### > Метод `addNewDish`:
>    + Создаётся список  `dishesForType` который будет хранить список блюд
>    + Когда пользователь вводит тип, то начинается проверка с помощью `.containsKey` на наличие
>    + Если присутствует добавляем блюдо в этот тип - в переменную списка, а значит `dishesForType = dinnersByType.get(dishType)`
>    + Если нет - создаём новый тип (создаем новый объект списка `dishesForType`), а дальше добавляем в коллекцию с помощью метода `.put`
>    + В независимости от того новый список или нет, добавляем с помощью метода `.add` блюдо в список `dishesForType`
>    + создаётся переменная типа `String` - `dishType`, для ввода типа 
>    + создается переменная типа `String` - `dishName` для ввода названия 
>    + Так же вызывается метод `addNewDish` (с аргументами - `dishType` `dishName`) который отвечает за логику добавления Типа и Названия блюд в меню.
### > Метод `generateDishCombo` - содержит логику по генерации комбинаций блюд:
>    + Запрашивает количество наборов, и сохраняет в переменную `numberOfCombos` 
>    + Запрашивает (Типы) блюд, из которых будет состоять комбинация 
>    + Далее создается новый список который будет хранить введенные пользователем блюда `selectedTypes`
>    + Реализуем проверку с помощью `while` и метода `.isEmpty` (который проверяет ввод на пустые строки)
>    + Для условия вызываем метод `checkType` из класса `DinnerConstructor` тем самым проверяем введенный тип на наличие в списке (логика метода: с помощью метода `.containsKey` ищем значение в коллекции, если значение найденное возвращаем `true`, так же сам метод принимает в себя аргумент `nextItem`, и содержит параметр для хранения копии адреса `String type`)
>    + Если `true` - добавляем в список с помощью `.add` - веденный пользователем вариант, иначе выводим сообщение, что такого блюда нет.
>    > + Продолжение в приложении 2.
